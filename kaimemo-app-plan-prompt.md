あなたは
・クラウドアーキテクト
・シニアバックエンドエンジニア
・モバイルアプリ設計者
・DevOps設計者
を兼ねる技術責任者です。

以下の条件で「買い物リストWeb + Expoスマホアプリ」の
現実的かつ無駄のない開発計画を作成してください。
抽象論は禁止。
必ず実装レベル・運用レベルまで具体化してください。

---

# 【最重要制約】

1. AWS無料枠内で収まる設計を最優先すること
2. 段階的実装（MVP → 機能追加）とすること
3. 各フェーズごとに「実機確認 → 問題なければ次へ進む」方式
4. 環境構築は必要なタイミングで段階的に行う
5. 初期段階で過剰なインフラを作らない
6. 将来的にスケール可能な構造にする
7. アナリティクスは今回は未実装

---

# 【AWS無料枠制限条件】

- EC2: t2.micro / t3.micro 750時間以内
- RDS: db.t3.micro 750時間以内
- S3: 5GB以内
- データ転送：無料枠内
- Cognito: 無料枠内ユーザー数想定（少人数）
- CloudWatch最小利用
- NAT Gatewayは使わない（コスト回避）
- 可能なら単一AZ構成（初期）
- WebホスティングはS3 + CloudFront（AWS無料枠・無料利用枠内）を採用すること
- AWS Amplify Hostingは使わない

無料枠を超える可能性がある箇所は必ず明示すること。

---

# 【技術スタック確定事項】

※技術スタックはオーナーが詳しくないため、
　各カテゴリで最も適切・実績ある選定をしてください。
　以下は確定済みまたは方針として固定した項目です。

## 確定項目

- DB：RDS PostgreSQL（db.t3.micro）
- 通知連携：LINE Messaging API（LINE Botチャンネル使用）
- 音声入力：Web Speech API（ブラウザ標準・完全無料）優先。
　　　　　　スマホアプリはExpo Speech Recognition（無料）を使用。
　　　　　　AWS Transcribeは使わない（コスト発生のため）。
- 画像認識：AWS Rekognition（無料枠：月5,000画像まで）を使用。
　　　　　　無料枠を超える可能性がある場合は必ず明示すること。
- ジオフェンス：Expo TaskManager + 端末GPS（完全無料）を使用。
　　　　　　　　AWS Location Serviceは使わない（コスト発生のため）。
- アクセストークン：1時間（自動更新）/ リフレッシュトークン：14日（Cognito設定）
- Webホスティング：S3 + CloudFront
- CI/CD：GitHub Actionsを全フェーズで導入する。
　　　　　ただし初期フェーズは最小構成（lint + ビルド確認のみ）とし、
　　　　　フェーズが進むにつれてテスト・デプロイ自動化を段階追加していく。

## 技術選定をClaudeに委ねる項目（最適なものを選んで明示・理由も記載）

- バックエンドフレームワーク（Node.js前提：Express / Fastify / NestJS から選定）
- ORM（Prisma / TypeORM / Drizzle から選定）
- フロントエンドの状態管理ライブラリ
- オフライン同期ライブラリ（Expo向け）
- バーコード読み取りライブラリ（Expo向け）
- プッシュ通知基盤（Expo Push Notifications or AWS SNS）
